# 임베딩 모델 비교 정리

## 개요

임베딩 모델은 텍스트를 숫자 벡터로 변환하는 모델이다. RAG 파이프라인에서 문서 청크와 질문을 동일한 벡터 공간에 매핑하여 유사도 검색이 가능하게 한다. 모델 선택은 언어 지원 범위, 벡터 차원 수, 비용, 속도에 따라 달라진다.

## 각 모델 상세

### BAAI/bge-m3

- **제공**: BAAI (Beijing Academy of Artificial Intelligence), HuggingFace 무료 배포
- **벡터 차원**: 1024
- **특징**: Dense, Sparse, ColBERT 세 가지 검색 방식을 하나의 모델로 지원하는 멀티 기능 모델
- **장점**
  - 한국어 + 영어 혼용 문서에서 압도적인 성능
  - 100개 이상 언어 지원
  - 완전 무료 (로컬 실행)
  - ColBERT 방식으로 하이브리드 검색 내장 지원
  - 최대 8192 토큰까지 처리 가능
- **단점**
  - 모델 크기가 커서 로컬 GPU/CPU 자원 필요
  - 초기 모델 다운로드 시간 소요
- **추천 상황**: 한/영 혼용 문서, 로컬 실행, 무료로 고성능을 원할 때

---

### OpenAI text-embedding-3-small

- **제공**: OpenAI API
- **벡터 차원**: 1536 (dimensions 파라미터로 축소 가능)
- **특징**: OpenAI의 경량 임베딩 모델로 성능 대비 비용이 저렴함
- **장점**
  - API 호출만으로 사용 가능 (로컬 자원 불필요)
  - 성능 대비 가격이 저렴
  - 벡터 차원 축소 기능으로 저장 비용 절감 가능
  - 영어 성능 우수
- **단점**
  - API 비용 발생 (사용량 비례)
  - 한국어 성능이 bge-m3보다 낮음
  - 인터넷 연결 필요, 데이터가 OpenAI 서버로 전송됨
- **추천 상황**: 영어 문서 위주, 로컬 자원이 부족할 때, 빠른 프로토타입

---

### OpenAI text-embedding-3-large

- **제공**: OpenAI API
- **벡터 차원**: 3072 (dimensions 파라미터로 축소 가능)
- **특징**: OpenAI의 최고 성능 임베딩 모델
- **장점**
  - 영어 기준 최고 수준의 임베딩 품질
  - 벡터 차원 축소 기능 지원
  - 복잡한 문서 구조에서도 안정적
- **단점**
  - text-embedding-3-small보다 비용이 높음
  - 한국어 성능은 bge-m3에 미치지 못함
  - 데이터가 OpenAI 서버로 전송됨
- **추천 상황**: 영어 문서에서 최고 품질이 필요한 경우, 비용 여유가 있을 때

---

### multilingual-e5-large

- **제공**: Microsoft, HuggingFace 무료 배포
- **벡터 차원**: 1024
- **특징**: Microsoft가 개발한 다국어 임베딩 모델
- **장점**
  - 100개 이상 언어 지원
  - 무료 로컬 실행 가능
  - 영어와 다국어 균형 잡힌 성능
- **단점**
  - 한국어 성능이 bge-m3보다 낮음
  - bge-m3 대비 기능이 적음 (단일 Dense 방식)
- **추천 상황**: 영어 위주 다국어 문서, bge-m3의 대안으로 가볍게 사용할 때

---

### Ko-SBERT (Korean SBERT)

- **제공**: KLUE, HuggingFace 무료 배포
- **벡터 차원**: 768
- **특징**: 한국어에 특화된 Sentence-BERT 기반 임베딩 모델
- **장점**
  - 한국어 단일 언어 문서에서 높은 정확도
  - 무료 로컬 실행
  - 모델 크기가 작아 경량 환경에서도 실행 가능
- **단점**
  - 영어 성능 매우 낮음
  - 한/영 혼용 문서에 부적합
  - bge-m3 대비 전반적인 성능 낮음
- **추천 상황**: 한국어 전용 문서만 처리할 때, 매우 경량 환경

---

## 선택 가이드

```
한/영 혼용 문서이고 무료로 고성능을 원한다
└─ BAAI/bge-m3

영어 문서 위주이고 로컬 자원이 없다
└─ text-embedding-3-small (비용 절감)
└─ text-embedding-3-large (최고 품질)

한국어 전용 문서이고 경량 환경이다
└─ Ko-SBERT

다국어 문서이고 OpenAI 비용을 피하고 싶다
└─ multilingual-e5-large
```

## RAG 프로젝트 추천 조합

| 상황                       | 추천 모델              | 이유                                    |
| -------------------------- | ---------------------- | --------------------------------------- |
| 한/영 혼용 (현재 프로젝트) | **BAAI/bge-m3**        | 한국어 최강, 무료, 하이브리드 검색 지원 |
| 영어 문서 API 방식         | text-embedding-3-small | 비용 효율적                             |
| 영어 문서 최고 품질        | text-embedding-3-large | 최고 성능                               |

> **마이그레이션 팁**: 임베딩 모델을 변경하면 기존 벡터를 전부 재생성해야 한다. 처음부터 모델을 신중하게 선택하고 변경을 최소화하는 것이 중요하다.

---

## 벡터 차원과 성능의 관계

벡터 차원이 높을수록 더 많은 의미 정보를 담을 수 있어 검색 품질이 향상되지만, 저장 공간과 검색 속도에 영향을 준다. OpenAI 모델은 `dimensions` 파라미터로 차원을 줄여도 성능 저하가 적은 편이다. bge-m3의 1024차원은 성능과 효율의 균형이 잘 잡혀 있다.
