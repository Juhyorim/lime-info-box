# VectorDB 비교 정리

## 개요

VectorDB는 벡터 임베딩을 저장하고 유사도 검색을 수행하는 데이터베이스다.
RAG 파이프라인에서 임베딩된 청크를 저장하고, 질문 벡터와 가장 가까운 청크를 검색하는 역할을 담당한다.

## 각 VectorDB 상세

### Chroma

- **실행 방식**: 로컬 인프로세스 또는 별도 서버 모드
- **특징**: 설치와 사용이 가장 간단하여 프로토타입 및 사이드 프로젝트에 최적
- **장점**: 코드 몇 줄로 바로 시작 가능, LangChain과 완벽 호환, Python 친화적
- **단점**: 대규모 데이터에서 성능 한계, 하이브리드 검색 미지원, 프로덕션 사용에는 부적합
- **추천 상황**: 개인 프로젝트 초기, 로컬 테스트, 빠른 PoC

---

### Qdrant

- **실행 방식**: Docker 기반 로컬 실행 또는 Qdrant Cloud
- **특징**: 성능과 기능의 균형이 뛰어난 오픈소스 VectorDB
- **장점**: 하이브리드 검색(벡터 + BM25) 내장 지원, 고성능, Docker로 어디서든 실행 가능, 클라우드 이식성 우수
- **단점**: Chroma보다 초기 설정이 복잡, 직접 인프라 관리 필요
- **추천 상황**: 로컬 → 클라우드 마이그레이션 예정인 프로젝트, 하이브리드 검색이 필요한 경우

---

### Pinecone

- **실행 방식**: 클라우드 전용 (SaaS)
- **특징**: 완전 관리형 서비스로 인프라 관리 불필요
- **장점**: 관리가 매우 편리, 대규모 데이터에서도 안정적, REST API 제공
- **단점**: 클라우드 락인 리스크, 무료 티어 용량 제한, 오픈소스 아님, 비용 증가 가능성
- **추천 상황**: 인프라 관리 없이 빠르게 서비스 배포가 필요한 경우

---

### pgvector

- **실행 방식**: PostgreSQL 확장 플러그인
- **특징**: 기존 PostgreSQL DB에 벡터 검색 기능을 추가하는 확장
- **장점**: 기존 RDB와 벡터 검색을 한 DB에서 관리 가능, Spring Boot / JPA와 매우 친화적, SQL 문법 그대로 사용
- **단점**: 순수 VectorDB 대비 성능 낮음, 대규모 벡터 검색에는 한계
- **추천 상황**: 이미 PostgreSQL을 사용 중인 프로젝트, Spring Boot 백엔드와 연동, 기존 데이터와 벡터를 함께 관리할 때

---

### FAISS

- **실행 방식**: 로컬 인메모리 (디스크 저장 별도 구현 필요)
- **특징**: Meta(Facebook)가 개발한 고성능 벡터 유사도 검색 라이브러리
- **장점**: 소규모~중규모 데이터에서 검색 속도 최고, 무료, 경량
- **단점**: 영속성 없음 (서버 재시작 시 데이터 사라짐), 관리 기능 없음, 단독 사용 시 운영 어려움
- **추천 상황**: 검색 속도가 최우선인 연구/실험 환경, 다른 DB와 조합하여 캐시 레이어로 활용

---

## 선택 가이드

```
지금 당장 빠르게 시작하고 싶다
└─ Chroma

로컬에서 시작하고 나중에 클라우드로 올릴 예정이다
└─ Qdrant (Docker 기반으로 그대로 이식 가능)

인프라 관리 없이 바로 서비스로 배포하고 싶다
└─ Pinecone

Spring Boot / PostgreSQL 기반 백엔드에서 연동하고 싶다
└─ pgvector

검색 속도가 최우선이고 실험적으로 쓰고 싶다
└─ FAISS
```

## RAG 프로젝트 추천 조합

| 단계                | 추천 VectorDB | 이유                         |
| ------------------- | ------------- | ---------------------------- |
| 로컬 개발           | Chroma        | 설치 간단, 빠른 시작         |
| 클라우드 이전 후    | Qdrant        | Docker 이식, 하이브리드 검색 |
| Spring Boot 연동 시 | pgvector      | JPA 친화적, SQL 통합 관리    |

> **마이그레이션 팁**: Chroma → Qdrant 전환 시 임베딩 모델을 동일하게 유지하면 벡터 재생성 없이 DB만 교체할 수 있다.
