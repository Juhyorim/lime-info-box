# .gitignore 항목 설명

## `venv/`

Python 가상환경 디렉토리입니다.

- 수백~수천 개의 패키지 파일이 담겨 있어 **용량이 매우 큽니다** (보통 수백 MB)
- 운영체제 및 Python 버전에 따라 내용이 달라지므로 **다른 환경에서 그대로 사용할 수 없습니다**
- `requirements.txt`만 있으면 `pip install -r requirements.txt`로 언제든 재현 가능하기 때문에 굳이 올릴 필요가 없습니다

---

## `chroma_db/`

ChromaDB의 로컬 벡터 데이터베이스 저장 디렉토리입니다.

- 문서를 임베딩한 **실제 데이터(벡터값, 메타데이터 등)** 가 저장되어 있어 용량이 클 수 있습니다
- 업로드한 문서의 내용이 그대로 담길 수 있어 **개인정보 또는 민감한 데이터가 포함될 위험**이 있습니다
- 로컬 실행 환경마다 새로 생성되어야 하는 **런타임 산출물**이므로 코드 저장소에 적합하지 않습니다

---

## `uploads/`

사용자가 업로드한 PDF, Word 등의 파일이 저장되는 디렉토리입니다.

- 업로드된 파일에는 **개인정보나 기밀 문서**가 포함될 수 있습니다
- 사용자마다 다른 파일이 올라오는 **런타임 데이터**이므로 코드와 함께 관리할 필요가 없습니다
- 파일 용량이 커질 수 있어 git 저장소를 **불필요하게 무겁게** 만듭니다

---

## `__pycache__/`

Python이 `.py` 파일을 실행할 때 자동으로 생성하는 **바이트코드 캐시** 디렉토리입니다.

- Python 인터프리터가 **자동으로 생성·관리**하는 파일이므로 직접 관리할 필요가 없습니다
- Python 버전이나 실행 환경에 따라 내용이 달라져 **다른 환경에서 충돌**을 일으킬 수 있습니다
- 소스 코드(`.py`)만 있으면 자동으로 재생성되므로 **git에 올릴 이유가 없습니다**

---

## `.env`

환경변수 파일로, API 키나 설정값 등을 저장하는 파일입니다.

- API 키, DB 접속 정보, 비밀번호 등 **민감한 정보**가 담겨 있습니다
- git에 올라가면 **GitHub 등 공개 저장소에 그대로 노출**될 수 있습니다
- 대신 `.env.example` 파일에 **키 이름만 명시**하고, 실제 값은 각자 로컬에서 채우는 방식을 사용합니다

```
# .env.example
OPENAI_API_KEY=
DATABASE_URL=
SECRET_KEY=
```

---

## `*.pyc`

Python 소스 파일(`.py`)이 컴파일된 **바이트코드 파일**입니다.

- `__pycache__/`와 마찬가지로 Python이 **자동으로 생성**하는 파일입니다
- 소스 코드가 있으면 언제든 재생성되므로 **저장소에 올릴 필요가 없습니다**
- 운영체제나 Python 버전에 따라 달라져 **충돌을 유발**할 수 있습니다

---

## 요약

| 항목           | 제외 이유                                              |
| -------------- | ------------------------------------------------------ |
| `venv/`        | 용량 과다, 환경 의존적, `requirements.txt`로 대체 가능 |
| `chroma_db/`   | 민감 데이터 포함, 런타임 산출물                        |
| `uploads/`     | 개인정보 포함 가능, 런타임 데이터                      |
| `__pycache__/` | 자동 생성 파일, 환경 의존적                            |
| `.env`         | API 키 등 민감 정보 포함                               |
| `*.pyc`        | 자동 생성 파일, 환경 의존적                            |
